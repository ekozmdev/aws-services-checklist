# Repository Guidelines

## Project Structure & Module Organization
- 主要な成果物は `README.md` のチェックリスト表のみです。
- 余計なファイルやディレクトリは追加しません。

## Checklist Workflow
- ユーザーからコメントが来たら `README.md` の表のみを更新します。
- 表の列順は次の順番を厳守: Category, Service Name, Known, Touched, Notes。
- 「知っている」「触ったことがある」は `はい` / `いいえ` で統一します。
- メモは 1〜2 文で短く、学びや感想に絞ります。
- サービスはカテゴリごとにまとめ、可能ならカテゴリ内でサービス名のアルファベット順に整理します。

## Commit & Pull Request Guidelines
- コミットメッセージは Conventional Commits を使います（例: `docs: add checklist row for EC2`）。
- コミットは 1 変更 1 目的で小さく保ちます。
- PR には短い要約と関連リンク、必要に応じてスクリーンショットを含めます。

## Agent-Specific Instructions
- 更新対象は `README.md` の表のみです。
- 新規ファイルは作成しません。
- 更新後は行の整合性と表記ゆれを確認してから完了します。
